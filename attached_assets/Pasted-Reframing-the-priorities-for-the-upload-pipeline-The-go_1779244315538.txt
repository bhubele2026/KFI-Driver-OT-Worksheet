Reframing the priorities for the upload pipeline. The goal is NOT "fast uploads" — it's "every customer's file always finishes successfully, even big ones, paid once via the schema cache, slow-but-reliable on first contact."
Four asks in order. Don't move to #2 until #1 is solid, etc.
1. Fix the Adient truncation that's blocking real uploads. For files where the structure cache detects block-structured layout, drop per-chunk row count from ~118 to ~60. Yes, it doubles the chunk count and makes Adient take twice as long. That's acceptable. A 5-minute successful Adient upload beats a 2-minute failed one.
2. Switch chunk output format to NDJSON. One JSON object per line, no wrapping array. When a chunk truncates, every line before the cut is recoverable verbatim. Re-issue only the missing rows. This retires the entire chunk-halving-retry path (you flagged this as a #296 simplification earlier — now's the time). The Adient failure I just sent was salvage choking on a parse edge case; NDJSON makes that class of failure impossible.
3. Resume-from-checkpoint. As each chunk's rows successfully extract, persist them to a staging table keyed by upload ID. On retry of a failed upload, skip chunks already complete; resume from the first incomplete chunk. Critical for big-file first contacts where any single chunk failure currently wastes all prior work.
4. Verify the schema/recipe cache (readWithRoles) writes a reusable recipe on first Adient success. Second upload of Adient should be near-instant (TS-only, no Claude calls). If today's pipeline doesn't write a usable recipe on first contact, the "pay once" promise breaks. Add a test that simulates two consecutive uploads of the same fingerprinted file and asserts the second made zero Claude calls.
Out of scope for this round: targeted-driver extraction (we want full files), model-tier swaps (Sonnet stays), perceived-speed UI work (current "Reading 64s, Chunk 3 of 36" is good enough).
Goal: a brand-new customer with a 10,000-row file should upload successfully on first contact in 5-15 minutes, and every subsequent week's upload of that same format should finish in under 5 seconds.
